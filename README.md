Scope - This project designs a relational database system based on historical hotel booking data. The system supports cleaning and preparing booking records, analyzing relationships between lead time and price (ADR), and generating business insights to support predictive modeling and decision-making in the hospitality industry.

The database will store structured information about hotels, guests, bookings, rooms, and stay details. The system will allow managers and analysts to evaluate booking trends, cancellation patterns, pricing strategies, and seasonal demand behavior.

To satisfy database complexity requirements, the scenario extends the dataset into a realistic operational booking system while remaining within the hotel booking domain.

The system I will make suspports
-Cleaning booking data
-Structured hotel booking record management
-Time relationships, price, and cancellation behaviour
-Predictive modeling for hotel industry.

Goal - This project aims to analyze historical booking data to find trends and patterns to build a predictive model to support decision making in the hotel or hospitality industry. Evaluating price strategies and studdying seasonal demand patterns to see the resort  and city hotel performances.
